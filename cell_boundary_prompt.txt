
STEP 1: Setup

Write a single HTML file based JS  application.
Save it in "cell_boundary_v1.html" file.
This app models a small LTE network with four cells: N,W,S,E with ovelapping coverage.
For example, N,W,S,E located in the North, West, South, East and are pointed towards the center.
Make carrier for each cell configurable: it will be either 800 MHz of AWS.
By default Assume N,W are 800 MHz carrier cells, S,E are 800 MHz carrier cells.

Place the network in Plano Texas. 
Use leaflet open street map as a basemap to make the demo more relistic.
Add a check mark widget for sitching the leaflet map off.

The network has M stationary LTE UE devices places randomly.
Make M configurable.
Each UE has coordinates (x,y) and a dot showing it on the map. 
Make the dot size configurable.

Compute RSRP signal of each UE to all four cells. 
Add a random factor R to the RSRP value.
Choose the strongest signal and connect to the corresponding cell.
Use color coding red, blue, green, cyan for each UE connected to N,W,S,E , respectively.

make background light yellow. 
show sectors =off by default.
default UE count 1000. Max UE count 5000. Min UE count =10.
default dot radius=3. Min dor radius =1. max UE radius = 6.



Step 2

Add a button STEP2 for executing the following code.

Select a cell A=N. We will compute it's effective boundary in a few steps.
Look at all UE RSRP measurements: some will be connected to A=N, 
The rest of the UEs will be connected to cells which are not A (ie W,S,E).
We will denote them by O = UEs connected to other cells and use black color for them
Use back color for those points corresponding to O on the map.

Generate a GxG grid over the coverage area. Make G configurable. 
For each grid cell compute average of nearby RSRP of nearby locations for A and O. 
In the center of that cell put either color of A or O depending on the strength of the avarage RSRP signals.
Show the new map on another canvas titles STEP2.
